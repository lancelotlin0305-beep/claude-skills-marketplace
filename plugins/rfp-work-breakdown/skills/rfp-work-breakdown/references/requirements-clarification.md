# 需求釐清紀錄(疑義澄清)

RFP / 招標文件常有模糊、前後矛盾、語意不明的條文。把這些疑義逐筆追蹤到業主確認,是防止需求走鐘的
防線。這是**與工項 MD 分離的獨立交付物**,套 `assets/clarification-log-template.md`。

## 用法

- 拆工項時,`rfp-breakdown.md` 收集到的疑義,逐筆登錄到本清單(獨立 MD),標註對應條文。
- 與業主開會 / 確認後,更新「業主答覆」與「狀態」。
