# Builder 內部機制規格(internals)

> **產圖不需讀本檔。** 這裡是 `bpmn_builder.py` 自動機制的設計規格,
> 供維護/除錯 skill 時查閱;產圖所需慣例與檢核判準都在 `conventions.md`。

## 自動佈局:多版面試算(取法 drawio-skill 的 --tune)

分層產生多個版面變體,各以 `layout_score` 評分取低者、同分取列數少者:
- ①**緊湊版**——同欄後繼與閘道分支才降列,跨欄順序交接(來源非閘道)
  同列並排(水平走線),沒有上下關係的交接不佔新列,壓低總列數、
  避免樓梯狀長圖。
- ②**傳統版**——每個前向邊一律降列(迴圈/合流較不易穿框)。
- ③**加寬版**(列數 > 12 才逐級加試,子欄 2 → 3,上限 3)——把長鏈所在
  泳道加寬為多子欄牛耕式蛇行(偶數列左→右、奇數列右→左,換列處
  垂直直下、絕不環繞跳欄):僅閘道分支降列,其餘交接一律同列並排;
  原 route 提示(針對單欄版面)於此版一律重設 auto 重新選路。
- **「過長」計入選型評分**:每超出 12 列計 4 分(交叉為 10)——減列是
  目標的一部分,加寬省 3 列以上時即使多一條交叉仍會中選。
- Collab 在**整體層級**試算(單 pool 各自取優會看不到跨 pool 訊息流的重合)。
- `auto_layout` 開頭快照使用者原始 route 提示(`_route_hints`),
  供修正輪每次試算重設乾淨重選。

## 選路(_auto_routes)

- 第一輪貪婪決策:邊界出邊固定 bndSide、回頭邊 backLoop、同欄受阻跳層
  左右試算取衝突少者(平手取右)、閘道分支「慣例側出 vs 正交下行」試算。
- **第二輪精修**:全景已知下逐條重試候選(straight/side/enterRight),
  消除「先手選了會與後手交叉」的殘餘;backLoop/bndSide 走專用通道不重選。
- 同一節點多條邊由**端口重用懲罰**自動分散進出點;conflicts 含
  「平行貼行(2~10px)×5」懲罰,引導路線遠離既有軌道。
- 選路取優次序:**衝突 → 轉折數 → 路徑長**。

### 端點選擇(對應 conventions「端點選擇總則」)
- 每節點四接點 top/bot/left/right(`ports()`);起訖點依目標相對方位取**最近端**:
  `_orth_down()` 目標在下走 bot→top、同列在右走 right→left、在左走 left→right。
- 最近端造成穿越/重疊時退**鄰近端**:改走 side 系列(sideLeft/sideRight 出側邊)
  或 enterRight(從右接入)。
- **不用對向端**(背對目標那面):對向端是「穿越自身節點」與「繞行過長」的共同
  上游——從此面出線必先繞過節點本體才能轉向,不是繞成非最短就是最短卻穿框。
  故從端點選擇階段即排除,比事後修繞線更根本;此性質已內建於 route 引擎,
  不另設獨立檢核(其違規後果由既有「連線穿越節點」「繞行過長」評分覆蓋,不會 silent)。

## 重合分軌(7-1 機制)

精修後仍共線的邊(排除端點 15px 樁段——同端口短樁屬「端口重合」職責,
不重複計分),把後決定那條的共線垂直段整段平移(≥12px)、平移併入
鄰接水平段(整段挪移,不插短橫移毛刺=視覺斷頭),持久化於
`wps_override` 供三輸出與檢核共用;隔不開者由評分回報。
分軌候選含框線避讓;關連線另有「端口讓位」:與流程邊撞點時沿節點
邊緣平移 14px 錯開(BPMN 慣例:關連可從邊上任意點進出)。

## 側通道(sideLeft / sideRight)

- 基準通道位:左緣 +14 / 右緣 -14。
- **側通道讓位(20260709.01)**:左廊有 k 條 bndSide 讓位軌(目標在左的
  邊界事件出邊)時,sideLeft 外移 k×LINE_GAP 落於讓位軌之外
  (`_stash_bnd_tracks()` 統計、節點 `_bndL` 傳遞;右緣 -14 與讓位軌
  -28 起跳設計時已錯開,毋須處理)。未加寬時左邊距 36px 僅容兩條
  讓位軌,讓位後空間不足由修正輪加寬提供。
- **多軌巢狀(深外淺內)**:共用側通道的重合對,分軌方向依目標深度
  定向——深者只許外移、淺者只許內移(反向必生入段交叉);單向失敗
  (貼框)即 fallback 改移對方。
- 同側**部分重疊跨距**(非巢狀)的側邊對為結構性交叉,由選路分邊
  (一左一右)化解;兩側皆不可行時殘餘由評分如實回報。

## 邊界事件讓位軌(bndSide,固定慣例不進試算)

先落入列間水平走廊(宿主底與下一列的空隙),沿泳道邊緣夾縫
(左 +12 起、右 -28 起,每軌 +LINE_GAP)垂直下行,從側面進入目標——
避開宿主正下方的扇出與既有側通道。同宿主多顆邊界依「圓的幾何序」
巢狀錯開(近讓位側=淺廊+外軌);邊界圓的排列依「出邊目標」語意排序
(目標在右者靠右;同向群內目標列淺者靠讓位側)——幾何序因此等同
目標深度序,交錯破口不再出現。不進候選試算的原因:專用走廊+夾縫的
幾何已保證同宿主巢狀不交叉,進試算反而因微小幾何差異引發全域路線
組合漂移(同分換序)而不穩定。

## 鐵則歸零機制之二(20260710.16,學自使用者三輪標紅)

- **閘道端口衝突消解**(_auto_routes 第二輪半之二):菱形豁免端口分散,
  進線×出線共用頂點時,可改路的邊(非 backLoop/bndSide)改走
  「未佔用頂點」的最佳候選(straight/side*/enterRight[防呆]/out*[僅出邊,
  方向防呆]),鐵則⑥優先於評分。
- **分邊↔消解兩輪收斂**:消解改路會在上游閘道生成新共點;
  `_force_split_gateway_pairs` 於消解前後各跑一次。
- **關連線走廊避讓 `_assoc_dodge`**:共線/長並行/穿節點時,中段水平→
  垂直段平移 ±LINE_GAP/±2×LINE_GAP;不足時**首段插折**(起點後 18px
  橫移 dd 再續行)並與走廊平移**二階組合**——端點段釘在夥伴節點上
  不能整段平移,夥伴正下方貼著節點時只有插折能解。

## 鐵則歸零機制(20260710.15,學自使用者二輪標紅)

- **enterRight 方向防呆 `_er_ok`**:目標在來源中線右側時禁用(末段須
  橫穿目標本體接右緣=對向端;衝突分豁免自身端點看不見此穿越)。
  三處把關:首輪候選、避讓 fallback(改側通道試算)、二輪精修。
- **穿自身端點檢核**:端點前/後 12px 落在自身節點內部即報
  「穿過節點…(對向端接入)」。
- **回線同心巢狀**:_assign_backloop_tracks 依 (T.row, -S.row) 排序
  ——最深來源佔最外(左)軌;_spread_ports 垂直側 _away 按鄰接點 x
  (左側升冪/右側降冪)排序,外軌自動配上位進點,同目標多回線不交叉。
- **同源閘道雙出邊強制分邊**(_auto_routes 第二輪半,鐵則優先於評分):
  共線或共出點的雙出邊試「雙側出/單邊側出×2」三組合,驗證互不重合、
  出點相異、側出不反向繞穿菱形、不穿其他節點才採納。**勿改回
  「把 outLeft/outRight 加入精修候選」**——全域波及會使無關圖多出交叉。
- **工件避開閘道列**:_place_artifacts 候選鍵插入 gwpen(閘道列
  降級)——閘道列是分支側出走廊,工件落此堵死分邊。

## 重疊/長並行防治(20260710.14,學自使用者標紅回饋)

- **檢核豁免修正**:_check_placed 重合/間距檢核原對「共端點邊對」整組
  豁免——同目標雙回線 720px 共線、同源閘道雙出邊 230px 共線全數漏檢。
  豁免移除;端樁短重合仍由 _wps_overlap 的 trim=15 排除。
- **長並行雙門檻**:_wps_par_close 增 minlen 參數;檢核端另跑
  (lo=2, hi=2×LINE_GAP, minlen=150)——長並行須 ≥24px。共端點對僅檢
  長並行(短距貼行為端口分散所致,屬正常)。
- **backLoop 逐軌**:_assign_backloop_tracks 依(目標列,來源列,fid)
  排序寫入來源節點 `_blt`,waypoints 左通道 x = 左緣+14+_blt×2×LINE_GAP。
- **端口分散長並行加距**:_spread_ports 成員鄰接段 >150px 時錯開間距
  升為 2×LINE_GAP。
- **關連線全體互覺**:assoc_waypoints 改為首呼叫時 _route_assocs 一次
  決定全部(後決定者計入先決定者的重合×8/交叉/貼行×5/端口×2 衝突分,
  端口讓位至多三次遞移);快取 _assoc_wps 隨 _auto_routes 失效。
  原本各條獨立選路彼此不可見,同夥伴多條關連線必然同路重合。
- **conflicts 重合權重 1→2**:原比平行貼行(5)還輕;實測 8 過重
  (貪婪決策被推出原本沒有的交叉),2 為平衡點。
- **loopsrc 最左**:迴圈親和分欄內,直接接回頭邊的節點排最左
  (含全迴圈列),回線出邊就近入左通道不橫穿同列節點。

## 迴圈親和分欄(chainL / wideL,20260710.13)

學自使用者手修 .drawio(NSRRC A-1:決標鏈整條移右欄、廢標鏈移左欄)。
auto_layout 先以 DFS 拓撲序標記 `loopish`(能沿前向邊走到回頭邊起點的
節點);chainL/wideL 於蛇行子欄指派時,對「迴圈鏈×終止鏈混列」的
(lane,row) 群組改用固定分欄:迴圈鏈自左起算(鏈內依 pos 反序,越近
迴圈出口越靠左)、終止鏈靠右對齊(kk-len(tm) 起),不做偶奇列交替;
純同類列維持原蛇行。群組成員數 > lane_k 時退回原邏輯(避免 want 撞欄)。
候選加入 auto_layout 與 Collab 兩處逐級加寬迴圈,由評分取優、不強制。

## 鐵則機制(20260710.12)

- **泳道最小寬度 `_lane_min_width`**:全泳道 ≥2×LANE_W、格點節點 >20 個
  的泳道 ≥3×,以 lane_pad 對稱補足(`_place` 的 pad//2 置中)。呼叫點:
  auto_layout 每個候選(`_try` 開頭重置 lane_pad)、定案版、_ensure 手動
  路徑。Collab 因此在 `_reflow_pools` 重算 pool 原點+重放置+自動池重選路
  (wps_override 為絕對座標,平移必須重產;手動池尊重使用者路線)。
- **端口分散 `_spread_ports`**(_auto_routes 末端):同矩形節點同側共用
  端點的順序流沿側錯開 ≤LINE_GAP、夾邊 8px,端點與鄰接折點同步平移保持
  正交,寫入 wps_override。直線邊(2 點)保原位;圓/菱形節點豁免
  (`_SPREAD_TS`)。
- **跨線橋**:`.drawio` 邊樣式 `jumpStyle=arc;jumpSize=10`(原生);
  SVG 由 `_hops_path(wps, segs)` 在與先畫線段的嚴格交叉處插入半圓弧
  (`A5,5`),`_svg_seqflows`/`_svg_assocs` 共用同 pool 的 segs 登錄,
  後畫者跳線;貼近端點/折點 r+3px 不畫橋。emit 以 svg 計數如實回報。
- **鐵則回報**:`IRON_KEYS` 分類 → `_print_page_report`(終端)與
  `_iron_md_section`(.md「鐵則檢核」區);交叉列提醒(已橋接)不列違規。
  `ENGINE_TIME_CAP=300`:每張圖引擎耗時超過即警示。

## 五輪修正(_repair_rounds)

**確定性(20260710.11,維護時勿退回)**:候選一律以 `sorted()` 迭代
(涉事節點 set、涉事泳道 set)。擇優為嚴格 `<`(平手取先者),候選順序
若隨 PYTHONHASHSEED 變動,同一定義會隨機落在不同版面
(實測 A-2 採購案在 55/70 分兩解間跳動);排序後任意環境重跑必得同一結果。

產出前若仍有交錯/重疊/間距/端口缺陷,對涉事節點嘗試變異並全量
重佈線,擇優採納至多五輪,無改善即停:
- 候選:與鄰列佔位者**交換** / 移到鄰列**空位** / 涉事泳道**河道加寬**
  (`lane_pad` +2×LINE_GAP)/ 每個換位候選同時試「**換位+加寬**」
  組合候選(換位與加寬常須同時成立才降分,逐一試算會全數落選——
  如邊界事件扇出佔滿左廊 + 同欄雙長跨線的情境)。
- 每次試算前路線一律重設回 `_route_hints` 快照全量重選(沿用舊定案
  + 逐條精修翻不出協同換邊的最佳分邊)。
- 加寬只發生在自動佈局;手動模式仍以 `dx` 微調,泳道維持單欄。

## 工件放置(_artifact_widen + _place_artifacts)

工件不參與分層網格。含工件夥伴的泳道由 `_artifact_widen` 無條件加寬
≥2 子欄(每個佈局候選一體評分;與 chain/wide 疊加取 max),
`_place_artifacts` 依「夥伴同泳道相鄰子欄(右優先)→ 同泳道他列 →
最後手段鄰泳道」排序放置;手動指定 row 者尊重。手動佈局下僅工件
省略 row 時亦走此自動放置(流程節點缺 row 才觸發整體自動佈局)。
原「跨泳道貼鄰+左鄰 backLoop 走廊懲罰」機制已於 20260710.09 移除
(工件不再跨泳道,懲罰失去對象)。

## 檢核實作參數(對應 conventions 檢核表)

- 平行間距:讓位軌距/走廊錯開/分軌位移已統一加寬至 ≥12~14px 配合
  (門檻 <MIN_LINE_GAP=10)。
- 框線重疊:容器 hull 齊格化(x 取泳道並集邊界內縮 6、y 避開折點帶;
  頂距 -50,標題帶 y0+4~22 讓出 _orth_down 折點帶)。
- 訊息流穿框為提醒級(跨 pool 飛越屬慣例,與檔案級 validator 對齊)。
- **順序流方向性(20260710.04)**:emitter 端 `_DIO_EDGE` 顯式輸出
  `endArrow=block;endFill=1`(關連 `_DIO_ASSOC` 維持 none、訊息流 `_DIO_MSG`
  維持 open);SVG 端以 `marker-end="url(#arr)"` 保證。此項為**檔案級硬性**:
  只在 `validate_bpmn.py` 座標再驗攔截(順序流 style 須含 endArrow≠none),
  **不進 check_layout / layout_score 加權**(幾何評分器不檢查樣式字串)——
  故 conventions 評分表列為「硬性」而非權重分,三方帳目一致。

## 事件標籤與版面輸出細節

- 事件圓 38px:≤4 字置圓內、超過移右側(每行 6 字換行)。三輸出同步:
  SVG 直接繪製;`.bpmn` 補 BPMNLabel DI(否則 bpmn.io 放圈外下方);
  `.drawio` 依字數選圈內/右側樣式。
- `.bpmn` pool DI 上緣高出泳道 36px 頂帶,參與者名稱不壓泳道標頭;
  `.drawio` 另有圖頂橫式標題(name+版號)與泳道頭底線,bands 時
  左直條省 pool 名(皆與 SVG 規則一致)。

## HTML 檢視器內嵌機制

- 內嵌**未補白** SVG(`build_svg(x, pad_aspect=False)`):檢視器自帶 fit
  縮放,內嵌補白版會使內容 fit 後更小(直式長圖可能縮到剩一成)。
- 內嵌 SVG 載入後由 JS 依 viewBox 釘住元素尺寸(1 CSS px = 1 viewBox
  單位),否則瀏覽器以容器寬渲染,fit 的 scale 疊上去二次縮小。
- 初始縮放 = `min(視窗寬/內容寬, 視窗高/內容高) × 0.95` 並置中;
  雙擊與視窗改變大小皆重置回 fit。
